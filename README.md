# Салон красоты

Конфигурация для автоматизации бизнес‑процессов салонов красоты на платформе 1С:Предприятие.
(Вме данные на скриншотах вымышленные, любые совпадения случайны)

## О конфигурации

**Наименование:** Салон красоты  
**Версия:** 2.0.0.1  
**Платформа:** 1С:Предприятие 8.3 (версия 8.3.18 и выше)  
**Библиотека стандартных подсистем (БСП):** 3.2.1  

## Назначение

Автоматизация ключевых бизнес‑процессов салона красоты:
* управление продажами услуг и товаров;
* учёт закупок и складских остатков;
* управление персоналом;
* контроль денежных средств;
* бухгалтерский учёт.

## Функциональные возможности

* **Продажи:** запись клиентов, реализация услуг и товаров, учёт заказов.
* **Покупки:** поступление товаров, управление поставщиками, контроль склада.
* **Персонал:** учёт сотрудников, должностей.
* **Денежные средства:** кассовые ордера, учёт движения наличных.
* **Бухучёт:** план счетов, регистр бухгалтерии, оборотно‑сальдовая ведомость.
* **Справочники:** номенклатура, контрагенты, договоры, склады, кассы, сотрудники, статьи затрат.
* **Сервис и настройки:** управление пользователями, дополнительные отчёты и обработки.

## Интеграция

Поддерживается интеграция с внешними системами через:

* **HTTP‑сервисы:**
  * `GET /user` — получение информации о пользователе (аутентификация);
  * `POST /customerOrders` — создание заказа клиента.

### Взаимодействие с мобильной конфигурацией (папка сфайлами BeautySalon_MobileApp, демоверсия)

Мобильная конфигурация дополняет основную систему и взаимодействует с ней через **REST API**.
Для корректной работы необходимо развернуть сервер и настроить локальный IP адрес.

#### Аутентификация

1. Пользователь вводит логин (для тестирования можно использовать, например "Администратор").
2. Мобильное приложение отправляет `GET‑запрос` (`/user`) с логином в формате **JSON**.
3. Основная конфигурация проверяет логин по справочнику **«Контрагенты»** (реквизит **«ПользователиМП»**) и возвращает ответ (200).
4. При успешной проверке создаётся пользователь в мобильном приложении и открывается рабочая форма.

#### Создание записи клиента

1. После заполнения формы и нажатия кнопки **«Записать»** мобильное приложение отправляет `POST‑запрос` (`/customerOrders`) с данными в формате **JSON**.
2. Основная конфигурация:
   * создаёт и проводит документ
3. Мобильная конфигурация:
   * дублирует операцию локально;
   * показывает пользователю сообщение об успешном выполнении.

## Технические требования

* Платформа: 1С:Предприятие **8.3.18** или выше.
* Режим управляемого приложения: **да**.
* Режим использования модальности: **использовать с предупреждениями**.

## Установка

1. Установите платформу 1С:Предприятие 8.3.18 или выше.
2. Создайте новую информационную базу.
3. Загрузите конфигурацию из файла.
4. Выполните обновление информационной базы.
5. Настройте параметры системы в подсистеме **«Сервис и настройки»**.
6. Зарегистрируйте пользователей и назначьте права доступа.
7. **После публикации основной конфигурации** обновите **IP‑адрес** в тонком клинете в константе
