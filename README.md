# Салон красоты

Конфигурация для автоматизации бизнес‑процессов салонов красоты на платформе 1С:Предприятие.
(Все данные на скриншотах вымышленные, любые совпадения случайны)

## О конфигурации

**Наименование:** Салон красоты  
**Версия:** 2.0.0.1  
**Платформа:** 1С:Предприятие 8.3 (версия 8.3.18 и выше)  
**Библиотека стандартных подсистем (БСП):** 3.2.1  

## Назначение

Автоматизация ключевых бизнес‑процессов салона красоты:
* управление продажами услуг и товаров;
* учёт закупок и складских остатков;
* управление персоналом и расчёт заработной платы;
* контроль денежных средств;
* бухгалтерский учёт.

## Функциональные возможности

* **Продажи:** запись клиентов, реализация услуг и товаров, учёт заказов.
* **Покупки:** поступление товаров, управление поставщиками, контроль склада.
* **Персонал:** учёт сотрудников, должностей
* **Денежные средства:** кассовые ордера, учёт движения наличных.
* **Бухучет:** план счетов, регистр бухгалтерии, оборотно‑сальдовая ведомость.
* **Справочники:** номенклатура, контрагенты, договоры, склады, кассы, сотрудники, статьи затрат.
* **Сервис и настройки:** управление пользователями, дополнительные отчёты и обработки.

## Интеграция

Поддерживается интеграция с внешними системами через:
* **HTTP‑сервисы:**
  * `GET /getUser` — получение информации о пользователе;
  * `POST /createOrder` — создание заказа клиента.
  * Мобильное приложение для данных http сервисов лежит в папке BeautySalon_MobileApp
  (доступно только запись клиента и поиск пользователя в основной базе, приложение демо_версия)

## Технические требования

* Платформа: 1С:Предприятие **8.3.18** или выше.
* Режим управляемого приложения: **да**.
* Режим использования модальности: **использовать с предупреждениями**.

## Установка

1. Установите платформу 1С:Предприятие 8.3.18 или выше.
2. Создайте новую информационную базу.
3. Загрузите конфигурацию из файла.
4. Выполните обновление информационной базы.
5. Настройте параметры системы в подсистеме **«Сервис и настройки»**.
6. Зарегистрируйте пользователей и назначьте права доступа.

## Права доступа

Основные роли:
* **ПолныеПрава** — полный доступ ко всем функциям.
* **АдминистраторСистемы** — администрирование системы.
* **Администрирование** — базовые права администратора.

## Дополнительные возможности

* подключение дополнительных отчётов и обработок;
* настраиваемые варианты отчётов;
* подключаемые команды;
* управление печатными формами;
* интеграция с внешними системами через **HTTP/REST API**.
